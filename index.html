ZeroPy
Do transistor
ao interpretador.

Cada camada construída à mão, sem atalhos.

Python parece mágica até você ver o que está por baixo. Este projeto desce todas as camadas — bits, bytes, memória, CPU, objetos, compilador — até construir um interpretador funcional do zero. Sem usar bin(), sem importar parsers prontos, sem chamar eval().

Ver demo: Bits & Bytes ↓ Ver no GitHub
As camadas do Python

Um interpretador é feito de camadas. O hardware executa tudo, mas você escreve Python. Entre uma coisa e outra, existem 6 camadas. Este projeto constrói cada uma do zero. Clique em qualquer nível.

Hardware
Nível 0 — bits, bytes, memória, CPU
⚡
v1_bits_e_bytes.py — binário, ASCII, UTF-8, operações bit a bit e shift. ✅ Completo.

Objetos
Nível 1 — PyObject, tipos, hash table
📦
v1_pyobject.py — PyObject, reference counting, INCREF/DECREF.

Por que antes do código? Em Python, tudo é objeto. Precisamos de onde guardar dados antes de executar.

v2_parser.py — constrói a AST a partir dos tokens.
O que é o frontend? Lê o texto Python e transforma em estrutura que o computador processa.

v1_compilador.py — percorre a AST e gera bytecodes.

Por que compilar? Bytecodes são mais rápidos de interpretar que texto puro.

v1_bits_e_bytes.py — Nível 0

A base de tudo. Conversão decimal ↔ binário, tabela ASCII, operações bit a bit que a CPU executa diretamente. Interaja com o código ao vivo.

O algoritmo divide o número por 2 repetidamente e coleta os restos. Lidos de trás para frente, esses restos formam o número binário.
01000001 ASCII: "A"
Passos da divisão por 2
noperaçãoquocienteresto (bit)
A CPU opera sobre os bits diretamente. AND: 1 onde os dois são 1. OR: 1 onde algum é 1. XOR: 1 onde são diferentes. NOT: inverte tudo.
a = 204 11001100
b = 170 10101010
Cada letra é um número. A = 65, a = 97, espaço = 32. ord('A') → 65, chr(65) → 'A'.
